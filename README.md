## Repository Structure

- `src/` - Source code files
- `docs/` - Documentation files
- `tests/` - Tests and test data
- `scripts/` - Utility scripts
- `.github/` - GitHub-specific configurations (workflows, issue templates, etc.)

## Branching Strategy

- `main` - Stable production-ready branch
- `develop` - Integration branch for features, staging area before release
- `feature/` - Branches for each new feature, merged into `develop`
- `bugfix/` - Branches for bug fixes, merged into `develop` or `main` depending on urgency
- `release/` - Branches for release preparation, merged into `main` and `develop`
- `hotfix/` - Branches for urgent fixes, merged into `main` and `develop`
